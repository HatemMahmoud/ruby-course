Philosophy

Principles

Designed to make programmers happy,
following the principles of
good user interface design

1. Principle of Conciseness

Good servants do a lot of work with short orders

2. Principle of Least Surprise

Things work the way a programmer would expect them to

3. Principle of Flexibility

Languages should express,
not restrict, human thought

Features

Ruby is...

An open source, dynamic,
multi-paradigm, interpreted programming language

Open Source

Dual-licensed under the Ruby License and a BSD License

Dynamic

Class, module and method definitions in a program are built at runtime

Dynamic - Metaprogramming

Writing code that
writes code

Dynamic - Monkey Patching

Classes are never closed!
You can always add methods to an existing class

Dynamic - Dynamic Typing

Type checking is performed at runtime

Dynamic - Duck Typing

An object's current set of methods and attributes determines the valid semantics

Dynamic - Duck Typing

If it walks like a duck
and quacks like a duck,
it must be a duck!

Multi-Paradigm

Doesn't constrain to a single programming mindset

Multi-Paradigm - Object-Oriented

Every value is an object:
classes, numbers, strings, true, false, nil

Multi-Paradigm - Object-Oriented

Every object has a class and every class has one superclass

Multi-Paradigm - Object-Oriented

Object is the root of the class hierarchy

Multi-Paradigm - Procedural

Methods defined outside classes become members of the Object class

Multi-Paradigm - Functional

Ruby supports
anonymous functions and closures using blocks

Multi-Paradigm - Reflective

A program can observe its own structure and behavior at runtime

Interpreted

Ruby interpreter examines the source code and executes it on the fly

Classes

Example

class MegaGreeter
  attr_accessor :names

  def initialize(names = nil)
    @names = names
  end
end

Example

class MegaGreeter
  def say_hi
    if @names.nil?
      puts '...'
    elsif @names.respond_to?('each')
      @names.each do |name|
        puts "Hello #{name}!"
      end
    else
      puts "Hello #{@names}!"
    end
  end
end

Example

class MegaGreeter
  def say_bye
    if @names.nil?
      puts '...'
    elsif @names.respond_to?('join')
      puts "Goodbye #{@names.join(', ')}."
    else
      puts "Goodbye #{@names}."
    end
  end
end

  • Write a method that determines whether a given string is a palindrome (reads the same backwards as forwards, ignoring case, punctuation, and non-word characters.)
  • You may use Rubular.com.

Assignment #3

  • Write a method that, for a given string, returns a hash whose keys are words in the string and whose values are the number of times each word appears, ignoring case and non-word characters.

Assignment #4

  • Write a method that takes a two-element list for a rock-paper-scissors game, where Rock beats Scissors, Scissors beats Paper, but Paper beats Rock.
  • If the number of players is not equal to 2, raise WrongNumberOfPlayersError
  • If either player's strategy is something other than "R", "P" or "S" (case-insensitive), raise NoSuchStrategyError
  • Otherwise, return the name and strategy of the winner. If both players use the same strategy, the first is the winner.
